## Обзор проекта
Данный проект представляет собой действующую программно-аппаратную модель интерфейса базового доступа (**BRI**) сети **ISDN**. Система эмулирует логику физического и канального уровней передачи данных по архитектуре **2B+D**, используя метод временного мультиплексирования (**TDM**).

Проект демонстрирует возможность интеграции разнородных служб (Triple Play) в едином цифровом канале:
* **B1 (64 кбит/с)**: Имитация оцифрованного голоса через потенциометр.
* **B2 (64 кбит/с)**: Телеметрические данные с ультразвукового датчика расстояния.
* **D (16 кбит/с)**: Внеполосная сигнализация для управления вызовом (кнопка).

## Технические характеристики
* **Метод мультиплексирования**: TDM (Time Division Multiplexing).
* **Скорость передачи**: Масштабированная до **9600 бит/с** для обеспечения наглядности и стабильности демультиплексирования.
* **Синхронизация**: Программная, по уникальному маркеру начала кадра `0xAF`.
* **Интерфейс**: Эмуляция четырехпроводной шины **S/T** в топологии Point-to-Point.

## Структура TDM-кадра
Передача данных осуществляется пакетами фиксированной длины (4 байта) в строго заданном порядке:

| Байт | Назначение | Значение / Источник |
| :--- | :--- | :--- |
| **0** | **Синхронизация** | Уникальный маркер `0xAF` |
| **1** | **Канал B1 (Голос)** | Данные АЦП (0–255) |
| **2** | **Канал B2 (Данные)** | Расстояние с датчика HC-SR04 |
| **3** | **Канал D (Сигн.)** | Статус кнопки (0x00 / 0x01) |

## Аппаратная часть
Система реализована на двух микроконтроллерах, выполняющих разные роли в сети ISDN:
1. **Terminal Equipment (TE)**: Терминальное оборудование (Передатчик).
2. **Network Termination (NT)**: Сетевое окончание (Приемник с LCD-дисплеем).

**Используемые компоненты**:
* Arduino Nano (2 шт.)
* LCD 1602 I2C Display
* УЗ-датчик HC-SR04
* Потенциометр и тактовая кнопка (+ 10кОм резистор)
* Так же парочка светодиодов и резисторов 220-560 Ом к ним

## Алгоритм работы приёмника (NT)
Приёмник работает по следующему циклу:
1. Ожидание появления байта `0xAF` в буфере.
2. Захват последующих 3-х байт как единого кадра.
3. Демультиплексирование потока и вывод значений B1/B2 на дисплей.
4. Анализ D-канала для управления светодиодной индикацией вызова.

## Результаты верификации
В ходе тестирования подтверждена полная **независимость каналов**: изменение данных в канале B1 (вращение потенциометра) не вызывает задержек или искажений в канале телеметрии B2 и не влияет на скорость срабатывания сигнализации в канале D.
